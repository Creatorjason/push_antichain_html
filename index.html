Introduction


Bitcoin: The First revolution

The Bitcoin network originally developed by Satoshi Nakamoto implores the use of the Blockchain technology, Bitcoin harmonizes it's distinct and seperate components borrowed from other fields such as crytography, distributed systems, gossip protocols, economics, mathematics and so on to form such holistic masterpiece. In my own term I call a masterpiece such as Bitcoin an example of "clever cannibalism"(cannibalism in this context involves using parts from different systems to create another system). The challenges of double spend, consensus, digital asset replication and to mention a few were solved with the innovative solutions of the Bitcoin creator, I must say this is a work of pure genius. But this innovation was not without it's drawbacks, from my research into web3 and its components, this was when I discovered what the internet truly is, the very act of having more than one computer communicate and exchange information over a network makes an internet, Bitcoin achieves this but with a key difference it's decentralized. Though from a different perspective it still conforms to the idea of an internet. One would ask since computers in a network shares information such as video, image, text files etc, what is shared on the Bitcoin network. Well the intent of it's creator was to create an alternative to traditional financial systems such as banks and enable the use of digital currencies without trusting a third party, the Bitcoin network shares a copy of the blockchain containing transactions made by participants of the network, so in this regard the network participants receives the copy of the blockchain as a it's major information. With all the initial requirements for the creation of a peer to peer network for the transferrence of digital assest satisfied, Satoshi's work was done and his statement heard, one would have thought this is the end but then came Ethereum.


Ethereum: The Second revolution

On the shoulders of Bitcoin, Ethereum stands. Vitalik Buturin took the first revolution a step further, remember from the above definition of an internet, the Bitcoin settled for financial use case, in doing so it shut out and limited severely other use case of it's internet, Vitalik saw this and worked on it, all application on the internet are built with code a computer can understand and execute, this new found internet Web3, needed to exhibit this property too to cater for other use cases. Eventually Smart Contracts were introduced, which is a piece of code running on the a blockchain. Web2 has an advantage which is the ability to run computation or execute tasks from a dedicated computer called a Server, thereby it's system architecture of client-server enables for decoupling and seperation of concerns, whereas on the Ethereum network, it's participants (nodes) are both client and server at the same time, this tradeoff comes with it's advantages and disadvantages. One disadvantage is since the programming language used to create it's smart contracts is turing complete, that means loops are supported, a malicioius user can grind the network to a halt by running an infinite loop. The Ethereum team thought of this and decided to make it economically expensive to do so by using Gas, which is basically paying for computational efforts, when the gas for the computation runs out the program stops and it's state is reverted. One would think since most blockchain store transactions, how can applications be built on such a network, following first principles the Ethereum team made smart contract own addresses but this time, it received payments to run computations, the blockchain records this transaction too as a valid one since it has an address. This innovative solution gave the Third Web the leap it needed, very soon other blockchains began to spring up. Initially I thought you had to create your own programming language such Script or Solidity but then I stumbled on Solana built with Rust majorly, and then it hit me that more can be achieved.

ANTICHAIN

An Antichain is type of peer to peer network that works in opposite of a Blockchain. It's ancestor is append only, an Antichain doesn't consists of blocks but rather cells.
Cells
Cells just like a nodes on a Blockchain network are participants on the Antichain. Each cell has the following:
  • Owner
  • Gatekeeper
  • Drain
  • Overload status indicator
  • Address
  • Path (cell ID)
  • Transaction Pool
  • And other metadata.
    • Anyone in the world can create a cell, every cell is in either of two states live or not live, all cells starts out in a not live state. The Antichain consist of active cells which carry out network activities, on "chain" each cell must be actively involved, as if they are not they face deactivation and lose of "rights". From a financial use case, let's take an example of how the cells works, Cell A creates a transaction that is bound to Cell B, pushes it to it's mempool and tells Cell B about it. For Cell B to be able to receive the payment, it performs "cell knocking" operation which involves supplying the right parameters to the gatekeeper of Cell A, if valid, the gatekeeper of Cell A generates a witness object which consists of a list of paths and a nonce. Cell B receives this, for instance a basic "ver" field will contain [path1, path2, path3, path4, nonce], the quantity of paths is dependent on the preference of the Cell owner and the amount of coin being sent. Since the Gatekeeper of a cell is responsible for randomly generating a list of paths and nonce, Note: A path is basically the ID of a cell, with the witness object Cell B can commence on a "witnessing journey", which is simply an action where one cell takes a witness object from another cell, goes to other cells whose path are listed in the witness object, having them divide, the nonce is divided by a random divisor from that cell the value returned is a "dinonce" which is sent to the cell that's on a "witnessing journey", while the divisor is sent to the originator of the witness object, in return the cell is able to download transaction data. The process continues until all cells in the witness object has returned a divisor and a "dinonce", the recipient cell gathers these "dinonce" and presents them back to the gatekeeper of the originating cell, if valid the transaction is pushed to drain and the accounts of both sender and receiver are updated accordingly.
